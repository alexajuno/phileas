You are analysing a memory search query. Your job is twofold:

1. Expand the query into alternative phrasings that would help a retrieval system find relevant memories.
2. Decide whether the query contains ambiguous references (pronouns, kinship terms, demonstratives) that can't be resolved by literal string matching.

Original query: "{query}"

Respond with ONLY a JSON object with this exact shape:

{{
  "queries": ["<phrasing 1>", "<phrasing 2>", "..."],
  "needs_referent_resolution": true | false,
  "pronoun_hints": ["<term>", "..."]
}}

Rules:
- `queries`: 3-5 alternate phrasings. Include synonyms, more specific / more general versions, related terms.
- `needs_referent_resolution`: `true` when the query uses a pronoun (she, him, they, them), a kinship or relationship term that could refer to multiple people (mom, bro, boss, chị, em, anh), or a demonstrative ("that person", "the woman from last week") without a concrete name. `false` when the query already contains a specific name or when it's not about a person at all.
- `pronoun_hints`: the specific pronoun or kinship tokens that need resolution. Empty list when `needs_referent_resolution` is `false`.

Examples:

Q: "what does Alex work on"
A: {{"queries": ["Alex projects", "Alex current work", "what is Alex building"], "needs_referent_resolution": false, "pronoun_hints": []}}

Q: "who is chị I mentioned"
A: {{"queries": ["which chị", "who is this chị", "who is the woman I mentioned"], "needs_referent_resolution": true, "pronoun_hints": ["chị"]}}

Q: "my last meeting with her"
A: {{"queries": ["recent meeting", "meeting with her", "last one-on-one"], "needs_referent_resolution": true, "pronoun_hints": ["her"]}}

Q: "phileas recall pipeline"
A: {{"queries": ["phileas recall", "memory recall in phileas", "phileas retrieval"], "needs_referent_resolution": false, "pronoun_hints": []}}
