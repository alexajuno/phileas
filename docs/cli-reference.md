# CLI Reference

All Phileas commands with usage, options, and examples.

## Global options

```
phileas --version    Show version number
phileas --help       Show help for any command
```

---

## phileas init

Interactive setup wizard. Creates the config file, downloads embedding and reranker models, and optionally configures an LLM provider.

```bash
phileas init
```

The wizard prompts for:
1. Data directory (default: `~/.phileas`)
2. LLM provider (`anthropic`, `openai`, `ollama`, or `skip`)
3. Model name (provider-specific default)
4. API key environment variable name

Re-running `phileas init` overwrites the existing config file.

## phileas remember

Store a memory.

```
phileas remember TEXT [--type TYPE] [--importance N]
```

**Arguments:**

| Argument | Required | Description |
|----------|----------|-------------|
| `TEXT` | yes | The memory text (1-2 sentences) |

**Options:**

| Option | Default | Description |
|--------|---------|-------------|
| `--type` | `knowledge` | Memory type: `profile`, `event`, `knowledge`, `behavior`, `reflection` |
| `--importance` | `5` | Importance score from 1 (trivial) to 10 (critical) |

**Examples:**

```bash
# Store a knowledge memory with default importance
phileas remember "I prefer Python for backend work"

# Store a high-importance profile memory
phileas remember "My name is Alex Chen" --type profile --importance 9

# Store an event
phileas remember "Started at Acme Corp in March 2026" --type event --importance 7

# Store a behavioral preference
phileas remember "I like detailed code reviews" --type behavior
```

**Memory types:**
- `profile` -- Who the user is: name, identity, core traits
- `event` -- Things that happened: dates, milestones, life events
- `knowledge` -- Facts and expertise: skills, preferences, opinions
- `behavior` -- Patterns and habits: workflows, communication style
- `reflection` -- Higher-level insights (typically generated by consolidation)

## phileas recall

Search memories by natural language query.

```
phileas recall QUERY [--top-k N] [--type TYPE]
```

**Arguments:**

| Argument | Required | Description |
|----------|----------|-------------|
| `QUERY` | yes | Natural language search query |

**Options:**

| Option | Default | Description |
|--------|---------|-------------|
| `--top-k` | `10` | Maximum number of results |
| `--type` | *all types* | Filter by memory type |

**Examples:**

```bash
# Broad search
phileas recall "what programming languages do I use"

# Limit results
phileas recall "work history" --top-k 5

# Filter by type
phileas recall "who am I" --type profile
```

The recall pipeline:
1. Expands the query using LLM rewriting (if configured)
2. Gathers candidates from keyword search (SQLite), vector search (ChromaDB), and graph traversal (KuzuDB)
3. Reranks with a cross-encoder model
4. Selects diverse results via MMR
5. Computes final scores from relevance, importance, recency, and access frequency

## phileas forget

Archive a memory so it is no longer returned in searches.

```
phileas forget MEMORY_ID [--reason TEXT]
```

**Arguments:**

| Argument | Required | Description |
|----------|----------|-------------|
| `MEMORY_ID` | yes | UUID of the memory to archive |

**Options:**

| Option | Default | Description |
|--------|---------|-------------|
| `--reason` | *none* | Reason for archiving (stored for audit trail) |

**Examples:**

```bash
phileas forget a1b2c3d4
phileas forget a1b2c3d4 --reason "no longer accurate"
```

Archived memories are kept in the database but excluded from search results.

## phileas update

Update a memory's text content in place.

```
phileas update MEMORY_ID SUMMARY
```

**Arguments:**

| Argument | Required | Description |
|----------|----------|-------------|
| `MEMORY_ID` | yes | UUID of the memory to update |
| `SUMMARY` | yes | New summary text |

**Examples:**

```bash
phileas update a1b2c3d4 "I'm a senior backend engineer who loves distributed systems"
```

The old version is archived as a snapshot and linked via a `SUPERSEDES` edge in the knowledge graph, preserving the full correction trail. The original creation date and identity are preserved.

## phileas list

Browse stored memories.

```
phileas list [--type TYPE] [--limit N]
```

**Options:**

| Option | Default | Description |
|--------|---------|-------------|
| `--type` | *all types* | Filter by memory type |
| `--limit` | `20` | Maximum number of items to show |

**Examples:**

```bash
# List all memories
phileas list

# List only profile memories
phileas list --type profile

# Show more results
phileas list --limit 50
```

## phileas show

Show full detail of a single memory.

```
phileas show MEMORY_ID
```

**Arguments:**

| Argument | Required | Description |
|----------|----------|-------------|
| `MEMORY_ID` | yes | UUID of the memory |

**Examples:**

```bash

```

## phileas ingest

Extract memories from a text file or raw text using LLM extraction. Requires an LLM to be configured.

```
phileas ingest SOURCE
```

**Arguments:**

| Argument | Required | Description |
|----------|----------|-------------|
| `SOURCE` | yes | File path or raw text to extract memories from |

**Examples:**

```bash
# Extract from a file
phileas ingest ~/notes/meeting-notes.txt

# Extract from raw text
phileas ingest "Had a great meeting with Sarah about the new API design. She prefers REST over GraphQL."
```

The LLM analyzes the text and extracts individual memories with types, importance scores, entities, and relationships. Each extracted memory is stored via the normal `memorize` pipeline, including deduplication.

## phileas consolidate

Find and merge clusters of similar memories into higher-level summaries. Requires an LLM to be configured.

```
phileas consolidate [--min-cluster N] [--max-clusters N]
```

**Options:**

| Option | Default | Description |
|--------|---------|-------------|
| `--min-cluster` | `3` | Minimum memories to form a cluster |
| `--max-clusters` | `10` | Maximum clusters to process |

**Examples:**

```bash
# Default consolidation
phileas consolidate

# Require larger clusters
phileas consolidate --min-cluster 5

# Process more clusters
phileas consolidate --max-clusters 20
```

Consolidation works on tier-2 memories (raw memories) and produces tier-3 memories (summaries). Clusters are found using vector similarity (threshold: 0.7 cosine similarity).

## phileas contradictions

Scan for conflicting memories. Requires an LLM to be configured.

```
phileas contradictions [--limit N]
```

**Options:**

| Option | Default | Description |
|--------|---------|-------------|
| `--limit` | `50` | Maximum memories to scan |

**Examples:**

```bash
phileas contradictions
phileas contradictions --limit 100
```

For each memory, the command finds related memories via recall and asks the LLM whether they contradict. Reports conflicting pairs with explanations.

## phileas export

Export all active memories as JSON.

```
phileas export [--format FORMAT] [--output PATH]
```

**Options:**

| Option | Default | Description |
|--------|---------|-------------|
| `--format` | `json` | Export format (currently only `json`) |
| `--output`, `-o` | *stdout* | Output file path |

**Examples:**

```bash
# Print to stdout
phileas export

# Write to file
phileas export -o ~/backup/memories.json
```

## phileas serve

Start the Phileas MCP server for AI client integration.

```
phileas serve
```

The server exposes memory tools (memorize, recall, forget, update, relate, about, timeline, profile, consolidate, status) over the Model Context Protocol. See [MCP Integration](mcp-integration.md) for client configuration.
